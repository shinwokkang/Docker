## 1. 비유로 이해하는 Docker: "항구의 화물선과 규격화된 컨테이너"

이해를 돕기 위해 무역 항구의 화물선을 상상해 보겠습니다.

### ❌ Docker가 없던 시절 (과거의 항구)
과거에는 배에 물건을 실을 때, 피아노, 과일 상자, 자동차, 액체 드럼통 등 모양과 크기가 제각각인 화물들을 그대로 실었습니다.
- **문제점:** 피아노 위에 무거운 드럼통을 올릴 수 없고, 과일 상자가 바닷물에 젖지 않게 따로 포장해야 합니다. 배를 옮겨 탈 때마다 짐을 내리고 다시 싣는 과정이 너무 복잡하고 오래 걸렸습니다.

### ⭕ Docker가 등장한 후 (현대의 항구)
이 문제를 해결하기 위해 **'표준화된 철제 컨테이너(Container)'**가 발명되었습니다. 물건이 무엇이든 상관없이 똑같은 크기의 네모난 컨테이너 박스 안에 넣고 문을 닫아버립니다.
- **해결책:** 배, 기차, 트럭은 그 안에 무엇이 들었는지 신경 쓸 필요 없이, 오직 **'규격화된 컨테이너'**만 옮기면 됩니다! 옮기기도 쉽고, 물건끼리 부딪혀 망가질 일도 없습니다.

이 비유에서 **배(Ship)**는 **우리의 컴퓨터나 서버(운영체제)**를 의미하고, **컨테이너 박스**는 바로 **Docker 컨테이너**를 의미합니다!

## 2. 왜 Docker가 필요할까요? (지옥의 매트릭스 해결)

강의에서 **"지옥의 매트릭스(Matrix from Hell)"**라는 무시무시한 용어가 나왔습니다. 개발자들의 현실적인 고충을 나타내는 말입니다.

### 개발자의 고충
하나의 웹 사이트를 만들려면 웹 서버, 데이터베이스(DB), 메시징 시스템 등 수많은 프로그램(소프트웨어)이 필요합니다.
- 내 컴퓨터(Mac)에서는 잘 되던 프로그램이, 동료의 컴퓨터(Windows)나 서비스가 돌아가는 실제 서버(Linux)에서는 **버전이 달라서, 설정이 달라서 에러를 뿜으며 실행되지 않는 일**이 일상다반사였습니다.
- 새로운 직원이 오면 컴퓨터 세팅하는 데만 며칠이 걸렸습니다.

### Docker의 구원
Docker는 **"프로그램을 실행하는 데 필요한 모든 파일, 설정, 라이브러리를 하나의 '컨테이너' 박스에 다 담아버리자!"**라는 아이디어입니다.
박스(컨테이너)만 똑같이 가져다 놓으면, 내 Mac이든 동료의 Windows든, 실전 서버든 **어디서나 100% 똑같이 동작**하게 됩니다.

## 3. 핵심 개념: 가상 머신(VM) vs Docker 컨테이너

컴퓨터 안에 또 다른 컴퓨터를 만드는 방법에는 두 가지가 있습니다. 이를 시각적으로 비교해 보겠습니다.

```mermaid
graph TD
    subgraph 가상 머신 (VM)
        A1[하드웨어] --> B1[하이퍼바이저]
        B1 --> C1[가상 머신 1: 무거운 OS + 앱]
        B1 --> C2[가상 머신 2: 무거운 OS + 앱]
    end

    subgraph Docker 컨테이너
        A2[하드웨어] --> B2[운영체제 OS]
        B2 --> C3[Docker]
        C3 --> D1[컨테이너 1: 가벼운 앱]
        C3 --> D2[컨테이너 2: 가벼운 앱]
    end
```

- **가상 머신 (VM):** 하드웨어 위에 완전히 새로운 운영체제(Windows, Linux 등)를 통째로 설치합니다. 무겁고, 느리고, 컴퓨터 자원(메모리, 용량)을 엄청나게 많이 차지합니다. (마치 집 안에 새로운 집을 지어버리는 것)
- **Docker 컨테이너:** 기존 운영체제의 핵심(커널)을 그대로 공유하면서, 딱 필요한 프로그램만 격리해서 실행합니다. **아주 가볍고, 1초 만에 실행되며, 자원을 적게 차지합니다.** (마치 집 안에 가벽을 세워 방만 나누는 것)

## 4. 이미지(Image)와 컨테이너(Container)

Docker를 배우면 평생 듣게 될 가장 중요한 두 단어입니다.

1. **이미지 (Image):**
   - **빵틀** 또는 **설계도**라고 생각하시면 됩니다.
   - 프로그램과 그 프로그램을 실행하기 위한 모든 설정이 꼼꼼하게 얼려 있는(저장된) 파일입니다.
   - 예: `붕어빵-틀`
2. **컨테이너 (Container):**
   - 빵틀(이미지)에서 찍어낸 **실제 빵**입니다.
   - 이미지를 바탕으로 메모리에 올라가서 **실제로 쌩쌩 실행되고 있는 상태**를 말합니다.
   - 예: `실제 먹을 수 있는 붕어빵`

## 5. 실습을 위한 준비 및 예고

이번 강의는 개요를 설명하는 시간이어서 본격적인 코드는 등장하지 않았습니다만, 강의에서 언급된 가장 중요한 명령어가 하나 있습니다. 바로 `docker run` 입니다.

### 📌 명령어 미리보기
앞으로 우리는 터미널(명령 프롬프트)에서 다음과 같은 명령어를 치게 될 것입니다.

```bash
docker run ansible
```

- `docker` : Docker야! 일해라!
- `run` : 컨테이너를 실행해줘!
- `ansible` : 'ansible'이라는 이름의 이미지(빵틀)를 가져와서!

즉, **"ansible이라는 이미지를 가져와서 컨테이너로 실행시켜라!"**라는 의미입니다. 단 이 한 줄만으로 수많은 설치 과정을 건너뛰고 프로그램이 실행되는 마법을 보게 될 것입니다.

### 🛠️ 실습 환경 세팅 안내
다음 시간에 본격적인 실습을 진행하기 위해 Docker를 설치해야 합니다.

1. **Mac용 Docker Desktop 설치:**
   - [Docker 공식 홈페이지(https://www.docker.com/products/docker-desktop/)](https://www.docker.com/products/docker-desktop/)에 접속합니다.
   - **'Download for Mac'** 버튼을 클릭하여 설치 파일을 다운로드합니다. (Mac의 칩셋이 Intel인지, Apple Silicon(M1, M2 등)인지 확인 후 알맞은 것을 다운로드합니다.)
   - 다운로드된 `.dmg` 파일을 열고 Docker 아이콘을 Applications 폴더로 드래그하여 설치합니다.
   - 설치가 완료되면 Docker 프로그램을 실행하고, 상단 상태 표시줄에 고래 아이콘 🐳 이 나타나면 성공입니다!

2. **터미널에서 확인하기:**
   - Mac의 `터미널(Terminal)` 앱을 엽니다.
   - `docker --version` 이라고 치고 엔터를 눌렀을 때, 버전 정보가 나오면 완벽하게 준비된 것입니다.
