## 1. 💻 Kode Kloud 실습 랩(Labs) 100% 활용하기

강사님은 여러분이 직접 터미널 환경을 구축하지 않아도, 브라우저에서 바로 실습할 수 있는 환경을 제공합니다.

### 랩(Lab) 환경 구성
- **왼쪽 화면 (터미널):** 우리가 앞서 연습했던 진짜 터미널 창입니다. 여기서 명령어를 자유롭게 칠 수 있습니다.
- **오른쪽 화면 (퀴즈 포털):** 미션(질문)이 주어집니다. "현재 설치된 Docker 버전을 확인하세요", "Redis 컨테이너를 실행하세요" 같은 미션이 나옵니다.

### 랩 진행 꿀팁 🍯
1. **힌트(Hint) 적극 활용하기:** 명령어가 기억나지 않으면 주저 없이 Hint 버튼을 눌러보세요. 처음엔 보고 치면서 손에 익히는 것이 중요합니다.
2. **Check 버튼으로 정답 확인:** 터미널에 명령어를 친 후, 오른쪽에서 `Check` 버튼을 누르면 정답인지 채점해 줍니다.
3. **🚨 주의사항 (보안 및 저장):** 이 환경은 **전 세계 누구나 접속할 수 있는 공개(Public) 환경**이며, **1시간 뒤에 모두 초기화(파괴)**됩니다. 절대 개인정보, 비밀번호, 중요한 소스 코드를 입력하거나 저장하지 마세요! 그저 마음껏 망가뜨리고 연습하는 놀이터로만 쓰시면 됩니다.

## 2. 💡 Chapter 2 총정리: 이 명령어들, 실무에서는 어떻게 쓰일까?

지금까지 배운 `run`, `ps`, `stop`, `rm`, `exec` 같은 명령어들, **현업 개발자들은 과연 언제, 왜 쓸까요?** 실무에서 가장 흔하게 마주치는 3가지 상황을 통해 이해해 봅시다.

### 🏢 상황 1: "새로운 프로젝트에 투입되었어요! 로컬 개발 환경 뚝딱 만들기" (`run -d`)
- **과거:** 내 컴퓨터(Mac)에 MySQL(데이터베이스), Redis(캐시), Nginx(웹 서버)를 일일이 설치하고 버전을 맞추느라 반나절이 다 갔습니다.
- **실무(Docker 활용):** 
  ```bash
  docker run -d mysql:8.0
  docker run -d redis:7.2-alpine
  ```
  명령어 단 두 줄만 치면 내 컴퓨터에 데이터베이스와 캐시 서버가 백그라운드(`-d`)로 순식간에 켜집니다! 개발자들은 굳이 무거운 DB를 설치하지 않고, 이렇게 컨테이너로 띄워두고 코딩을 시작합니다.

### 🏢 상황 2: "운영 서버에 에러가 났어요! 원인을 찾아라!" (`ps` 와 `exec -it`)
- **상황:** 갑자기 웹사이트가 안 들어간다는 연락을 받았습니다.
- **실무(Docker 활용):**
  1. 먼저 `docker ps`를 쳐서 웹 서버 컨테이너가 잘 살아있는지 확인합니다. (만약 죽어있다면 `docker ps -a`로 `Exited` 코드를 확인합니다.)
  2. 살아서 돌아가는데 에러가 난다면, **직접 컨테이너 안으로 들어갑니다.**
  ```bash
  docker exec -it 웹서버_컨테이너ID bash
  ```
  3. 컨테이너 내부로 접속한 뒤, 에러 기록(로그 파일)을 뒤져서 원인을 찾아냅니다. (`exec -it`는 개발자의 **청진기** 같은 역할을 합니다!)

### 🏢 상황 3: "퇴근하기 전, 컴퓨터 자원 청소하기" (`stop`, `rm`, `rmi`)
- **상황:** 여러 프로젝트를 동시에 하다 보니, 띄워둔 컨테이너가 너무 많아 Mac이 비행기 이륙 소리를 내며 느려졌습니다.
- **실무(Docker 활용):**
  ```bash
  # 1. 일 안 하고 있는 죽은 컨테이너들 한 번에 지우기
  docker rm 컨테이너1 컨테이너2 컨테이너3
  
  # 2. 당장 안 쓰는 프로젝트의 컨테이너들 잠시 멈춰두기
  docker stop 무거운DB컨테이너
  
  # 3. 용량만 차지하는 안 쓰는 설계도 지워서 디스크 용량 확보하기
  docker rmi 안쓰는_이미지
  ```
  이렇게 항구(내 컴퓨터)의 공간과 자원을 효율적으로 관리합니다.
